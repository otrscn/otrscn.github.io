参与协作

我们十分欢迎你参与到社区协作. 在加入我们之前请留意, 你需要签署 OTRS 协作协议 简称(CLA).简而言之就是要转让代码的所有权给 OTRS —关于这之前有过一个争论 , 见这个 博客. 作为开源项目来说, 此时比较好的解决办法就是先签署一份 CLA.请打印签上你的名字和 github 的用户名之后 扫描并发给我们.

这份 管理员手册 包含了很多信息帮助如何安装, 使用 和 调试 OTRS, 而这份 开发者手册(英文) 包含了详细的 OTRS 开发信息, 这份手册告诉你 如何搭建开发环境(英) 和 如何协作 OTRS (英).

在协作这一章还详细说明了有关 OTRS 代码风格指南, 请你预先阅读并遵守. 我们推荐使用这个 OTRSCodePolicy 模块来检查. 你也可以每次提交/注册之前进行手工检查之, 详细请参考该 模块 的说明, 通过 github 发送 PR 之前使用这个模块是必要的.

这里有一个 除错/分析 模块可以帮助开发人员: Fred. 它可以提供一些帮助, 例如数据库 SQL 日志输出, 错误日志输出, 基于 JavaScript 的 JSLint 代码校验以及 HTML 优化器.

如果有任何疑问, 请马上向 开发者邮件列表 发问.

稳定分支

当前维护的 OTRS 分支是 rel-5_0. 所有的 bug 修正将被合并到该分支持. 新的功能合并到稳定版本是不允许的, 而是加入到 Master 分支.

本地化 OTRS

翻译人员可以通过 Transifex 来本地化 OTRS. 我们简单的让你了解如何翻译 OTRS 服务台 和 OTRS 文档.
